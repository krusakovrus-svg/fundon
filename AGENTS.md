# AGENTS.md

## Общий режим работы

Этот проект развивается как отдельная новая версия продукта.

Главные правила:
- не копировать хаотично старые UI-решения из v1 без причины
- сохранять простую и читаемую архитектуру
- сначала строить базовую рабочую структуру, потом полировать визуал
- все решения должны быть production-friendly, а не concept-only
- мобильная версия — основной сценарий по умолчанию

## Технологический стек

Базовый стек по умолчанию:
- Next.js App Router
- TypeScript
- Tailwind CSS
- Framer Motion
- Supabase
- Vercel

## Архитектурные принципы

- маршруты должны быть тонкими и декларативными
- основная логика экранов должна жить в `src/components/...`
- общие visual wrappers должны жить в shared UI/layout слое
- данные, storage и интеграции должны жить в `src/lib/...`
- mock data и словари должны жить отдельно от UI
- не создавать тяжёлую абстракцию без реальной повторяемости

## Работа с кодом

- prefer small files and readable components
- typed props обязательны
- сложную логику выносить в hooks или helpers только когда это реально упрощает экран
- не смешивать server-data слой и browser-interaction слой в одном месте без необходимости
- локальный fallback должен оставаться простым и надёжным

## Языки

Все изменения по UI нужно делать сразу для обеих языковых версий:
- RU
- EN

Если явно не сказано иначе, русский должен рассматриваться как основная продуктовая локаль.

## Документация

После заметных архитектурных изменений обновлять документы в `documents/`.
